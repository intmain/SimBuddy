# SimBuddy — Definition of Done (DoD)

> 적용 범위: Phase 1 ~ Phase 4 전체

---

## 1. 테스트

- [ ] 핵심 로직에 대한 **Unit Test (XCTest)** 작성 및 통과
- [ ] 주요 사용자 흐름에 대한 **UI Test** 작성 및 통과
- [ ] 새로 추가/변경된 기능에 대해 기존 테스트가 깨지지 않을 것

---

## 2. 코드 리뷰

- [ ] PR을 생성하고 **셀프 리뷰** 완료 후 머지
- [ ] PR에 변경 사항 요약 및 테스트 결과 기록

---

## 3. 성능

| 항목 | 목표 | 최대 허용 |
|------|------|-----------|
| 메뉴 오픈 → 시뮬레이터 목록 표시 | 400ms 이내 | 1,000ms |
| 앱 목록 서브메뉴 로딩 | 400ms 이내 | 1,000ms |
| Storage Manager 용량 스캔 완료 | 400ms 이내 | 1,000ms |

- [ ] 목표치(400ms) 달성을 우선 시도
- [ ] 목표치 미달 시 최대 허용치(1,000ms) 이내이면 완료로 인정

---

## 4. UI / 디자인

- [ ] macOS 네이티브 룩앤필 준수 (시스템 기본 컴포넌트 활용)
- [ ] **UX 디자이너 에이전트**를 통해 디자인 생성 및 검수 통과

---

## 5. 문서화

- [ ] 코드 주석: 로직이 자명하지 않은 부분에 작성
- [ ] **README**: 프로젝트 소개, 빌드 방법, 사용법 정리
- [ ] **CHANGELOG**: Phase별 변경 이력 관리
- [ ] **아키텍처 문서**: 주요 모듈 구조 및 데이터 흐름 설명

---

## 6. 삭제 기능 안전장치

- [ ] 삭제 전 **확인 다이얼로그** 표시
- [ ] 다이얼로그에 **삭제 대상 목록 명시**
- [ ] 완전 삭제가 아닌 **macOS Trash(휴지통)로 이동**

---

## 7. 에러 핸들링

- [ ] 어떤 에러 상황에서도 **앱 크래시 없음**
- [ ] 에러 발생 시 사용자에게 **에러 메시지 표시**
- [ ] 에러 내용을 **로그 파일에 기록**

---

## 8. 빌드 / 배포

- [ ] Xcode 빌드 성공 및 로컬 실행 가능
- [ ] **CI 파이프라인** 통과 (자동 빌드 + 테스트)
- [ ] **배포 아티팩트 생성 자동화** 구성
- [ ] *(Nice to have)* 컴파일 경고 0개 — 에너지가 남을 때 처리

---

## 9. 접근성

- [ ] **VoiceOver** 기본 동작 확인 (주요 메뉴 항목 읽힘 확인)

---

## 10. Phase 완료 판정

- 위 DoD 항목을 **모두 충족**하면 해당 Phase 완료로 판정
- 별도 dogfooding 기간 없이 바로 다음 Phase 진행 가능
